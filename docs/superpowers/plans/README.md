# Phase plans (historical record)

Dated TDD planning documents for the earliest build phases — 0 (scaffold), 1a–1c (data spine),
and 2a–2d (backtest core). They are kept as project history and are not maintained.

Later phases (3–7: validation gauntlet, paper scaffold, tear sheet/CLI, broadening, Kronos) were
designed in [`../specs/`](../specs/) — one spec per subsystem — and tracked in the "Build status"
section of the repo-root `CLAUDE.md`; there are intentionally no plan files for them.
